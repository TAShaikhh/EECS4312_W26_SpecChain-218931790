# EECS4312_W26_SpecChain-218931790

This repository contains the course project artifacts for **MindDoc: Mental Health Support** (`de.moodpath.android`), using Google Play reviews as the source text for manual, automated, and hybrid requirements-engineering pipelines.

## Application Studied

- App: MindDoc: Mental Health Support
- Store page: <https://play.google.com/store/apps/details?id=de.moodpath.android&hl=en_CA&pli=1>
- Review source: Google Play Store
- Collection tool: `google-play-scraper`

## Final Dataset Size

- Raw dataset: `data/reviews_raw.jsonl` with 5,000 reviews
- Cleaned dataset: `data/reviews_clean.jsonl` with 4,278 reviews
- Dataset metadata: `data/dataset_metadata.json`

## Repository Structure

- `data/`: raw reviews, cleaned reviews, dataset metadata, and review groups for manual, automated, and hybrid pipelines
- `personas/`: personas for manual, automated, and hybrid pipelines
- `spec/`: requirements specifications for manual, automated, and hybrid pipelines
- `tests/`: validation tests for manual, automated, and hybrid pipelines
- `metrics/`: per-pipeline metrics and the cross-pipeline summary
- `prompts/`: saved automated prompt artifact used in Task 4
- `reflection/`: final project reflection
- `src/`: collection, cleaning, generation, metrics, validation, and workflow scripts

## Exact Commands

Run these commands from the repository root.

### Reproduce the automated pipeline

Set the Groq API key in the shell before running the automated workflow.

```powershell
$env:GROQ_API_KEY="YOUR_GROQ_API_KEY"
python src/run_all.py
```

This workflow runs:

```powershell
python src/01_collect_or_import.py
python src/02_clean.py
python src/05_personas_auto.py
python src/06_spec_generate.py
python src/07_tests_generate.py
python src/08_metrics.py --pipeline auto
```

### Recompute metrics for all pipelines

```powershell
python src/08_metrics.py --pipeline all
```

### Validate repository structure

```powershell
python src/00_validate_repo.py
```

## Reproducibility Notes

- `src/run_all.py` automates only the programmatic workflow required for the automated pipeline.
- Manual and hybrid artifacts are stored in the repository and are not regenerated by `run_all.py`.
- `src/00_validate_repo.py` checks that the required folders and project files exist and are readable.
